注意：此工程是在tinymp3-master压缩包解压后进行两点修改后的工程，以实现在ubuntu中gcc编译测试成功
	1：在main.c中的wavRead_int16函数里面的 drmp3_config pConfig; 后添加清空操作drmp3_zero_object(&pConfig);
		原因：新建该结构体后，内部成员不一定是零，非零的随机值会影响到后面的比特率与采样率的默认值赋值
			  功能里面是为零才赋默认值，不然就是一个随机值，而比特率和采样率会影响之后的转换算法，
			  算法涉及内存的申请分配，会导致死循环，使内存溢出，程序崩溃。
			  
	2：原工程默认 MP3是双声道转换的，本人测试从MP3--》wav--》MP3转换后会出现 新MP3比原MP3时长短一半，
		改成单声道转换后大大的降低了反复转换后导致的时长差
